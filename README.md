# Прогнозирование уровня воды в реке Кача (модель XGBoost)

Данный репозиторий содержит код для обучения модели градиентного бустинга (XGBoost), предназначенной для прогнозирования уровня воды в реке на 6, 24 и 72 часа вперед, а также для классификации степени паводковой опасности.

## Постановка задачи

Целью работы является разработка модели машинного обучения для краткосрочного и среднесрочного прогнозирования уровня воды. Это позволит заблаговременно выявлять риски наводнений и классифицировать их по категориям опасности.

Задача разделена на три подзадачи (три горизонта прогнозирования):
1. **Прогноз на 6 часов вперед.**
2. **Прогноз на 24 часа вперед.**
3. **Прогноз на 72 часа вперед.**

Каждая задача решается как задача регрессии (предсказание числового значения уровня воды в см). На основе полученного числового прогноза выполняется классификация состояния: "норма", "предупреждение" или "паводок".

## Данные и предобработка

Для обучения использовался синтезированный датасет, имитирующий гидрологические процессы. Для проверки способности модели к обобщению применялся второй датасет, выступающий в роли валидационного.

**Исходные признаки:**
- `datetime` - временная метка
- `temp_c` - температура воздуха (°C)
- `precip_mm` - количество осадков (мм)
- `humidity_pct` - относительная влажность воздуха (%)
- `wind_speed_mph` - скорость ветра (м/c)
- `pressure_hpa` - атмосферное давление (гПа)
- `water_level_cm` - текущий уровень воды (см)

**Инжиниринг признаков:**
- `water_level_trend_3d` - тренд изменения уровня воды за последние 3 дня (рассчитывается как разница между текущим уровнем и уровнем 12 записей назад)
- Целевые переменные (`target_6h`, `target_24h`, `target_72h`) создаются путем сдвига колонки `water_level_cm` вперед на 1, 4 и 12 строк соответственно

Все строки с пропусками удаляются. Числовые признаки стандартизируются с помощью `StandardScaler`.

## Модель

В основе решения лежит алгоритм градиентного бустинга **XGBoost (XGBRegressor)**. Выбор обусловлен его эффективностью при работе с табличными данными и способностью моделировать нелинейные зависимости.

**Оптимизация гиперпараметров:**
Для поиска наилучшей конфигурации модели применялся алгоритм **GridSearchCV** со следующими параметрами:
- `n_estimators`: [100, 200, 300]
- `max_depth`: [3, 4, 5, 7, 8]
- `learning_rate`: [0.01, 0.05, 0.1, 0.2]
- `subsample`: [0.7, 0.8, 1.0]
- `colsample_bytree`: [0.8, 0.85, 1.0]

Для корректной оценки на временных рядах применялась кросс-валидация `TimeSeriesSplit`. Обучение проводилось отдельно для каждого горизонта прогнозирования.

## Метрики

### Модель демонстрирует следующие показатели точности:

### Прогноз на 6 часов:
- MAE (средняя абсолютная ошибка): 4.42 см
- RMSE (среднеквадратичная ошибка): 6.01 см

### Прогноз на 24 часа:
- MAE: 4.41 см
- RMSE: 6.24 см

### Прогноз на 72 часа:
- MAE: 5.55 см
- RMSE: 7.86 см

Представленные метрики подтверждают практическую применимость разработанной модели для задач прогнозирования уровня воды.
